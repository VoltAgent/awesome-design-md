# Awesome DESIGN.md

将 DESIGN.md 复制到你的项目中，告诉 AI 智能体「按照这个风格帮我构建一个页面」，即可获得像素级还原、真正贴合设计规范的 UI。


## 什么是 DESIGN.md？

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是 Google Stitch 提出的全新概念。一份纯文本的设计系统文档，供 AI 智能体读取并生成风格统一的 UI。

它只是一个 Markdown 文件，无需导出 Figma、无需 JSON Schema、无需任何特殊工具。将它放到项目根目录，任何 AI 编程智能体或 Google Stitch 都能立刻理解你的 UI 应有的样子。Markdown 是大语言模型最擅长阅读的格式，无需任何解析或配置。

| 文件 | 读取方 | 定义内容 |
|------|--------|----------|
| `AGENTS.md` | 编程智能体 | 如何构建项目 |
| `DESIGN.md` | 设计智能体 | 项目的视觉风格与体验 |

**本仓库提供从真实网站中提取的、开箱即用的 DESIGN.md 文件。**

## 每份 DESIGN.md 包含什么

每份文件均遵循 [Stitch DESIGN.md 格式](https://stitch.withgoogle.com/docs/design-md/format/)，并包含扩展章节：

| # | 章节 | 内容说明 |
|---|------|----------|
| 1 | 视觉主题与氛围 | 情感基调、信息密度、设计理念 |
| 2 | 色彩方案与语义角色 | 语义命名 + 十六进制色值 + 功能用途 |
| 3 | 排版规则 | 字体家族，完整层级表 |
| 4 | 组件样式 | 按钮、卡片、输入框、导航栏及各状态 |
| 5 | 布局原则 | 间距比例、网格系统、留白哲学 |
| 6 | 层次与高度 | 阴影体系，界面层级关系 |
| 7 | 规范与禁忌 | 设计准则与反模式 |
| 8 | 响应式行为 | 断点、触控目标尺寸、折叠策略 |
| 9 | 智能体提示指南 | 快速色彩参考，开箱即用的提示语 |

每个网站包含：

| 文件 | 用途 |
|------|------|
| `DESIGN.md` | 设计系统文件（供智能体读取） |
| `preview.html` | 展示色块、字体比例、按钮、卡片的可视化目录 |
| `preview-dark.html` | 同上，深色界面版本 |

### 使用方法

1. 将对应网站的 `DESIGN.md` 复制到项目根目录
2. 告诉你的 AI 智能体使用它即可。
