# Awesome DESIGN.md

將 DESIGN.md 複製到你的專案中，告訴 AI 智能體「按照這個風格幫我建立一個頁面」，即可獲得像素級還原、真正符合設計規範的 UI。


## 什麼是 DESIGN.md？

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是 Google Stitch 提出的全新概念。一份純文字的設計系統文件，供 AI 智能體讀取並生成風格統一的 UI。

它只是一個 Markdown 檔案，無需匯出 Figma、無需 JSON Schema、無需任何特殊工具。將它放到專案根目錄，任何 AI 程式設計智能體或 Google Stitch 都能立刻理解你的 UI 應有的樣子。Markdown 是大型語言模型最擅長閱讀的格式，無需任何解析或設定。

| 檔案 | 讀取方 | 定義內容 |
|------|--------|----------|
| `AGENTS.md` | 程式設計智能體 | 如何建構專案 |
| `DESIGN.md` | 設計智能體 | 專案的視覺風格與體驗 |

**本儲存庫提供從真實網站中提取的、開箱即用的 DESIGN.md 檔案。**

## 每份 DESIGN.md 包含什麼

每份檔案均遵循 [Stitch DESIGN.md 格式](https://stitch.withgoogle.com/docs/design-md/format/)，並包含擴充章節：

| # | 章節 | 內容說明 |
|---|------|----------|
| 1 | 視覺主題與氛圍 | 情感基調、資訊密度、設計理念 |
| 2 | 色彩方案與語義角色 | 語義命名 + 十六進位色值 + 功能用途 |
| 3 | 排版規則 | 字型家族，完整層級表 |
| 4 | 元件樣式 | 按鈕、卡片、輸入框、導覽列及各狀態 |
| 5 | 版面原則 | 間距比例、網格系統、留白哲學 |
| 6 | 層次與高度 | 陰影體系，介面層級關係 |
| 7 | 規範與禁忌 | 設計準則與反模式 |
| 8 | 響應式行為 | 斷點、觸控目標尺寸、折疊策略 |
| 9 | 智能體提示指南 | 快速色彩參考，開箱即用的提示語 |

每個網站包含：

| 檔案 | 用途 |
|------|------|
| `DESIGN.md` | 設計系統檔案（供智能體讀取） |
| `preview.html` | 展示色塊、字型比例、按鈕、卡片的視覺化目錄 |
| `preview-dark.html` | 同上，深色介面版本 |

### 使用方法

1. 將對應網站的 `DESIGN.md` 複製到專案根目錄
2. 告訴你的 AI 智能體使用它即可。
